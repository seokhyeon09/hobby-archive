# 프로젝트 마무리 & README 작성


## 프로젝트 개요

제가만든 웹 사이트는 취미 공유 사이트이며, HTML, CSS 학습 내용을 시험해보기 위한 사이트입니다.

이번 실습에서 중점적으로 연습한 학습 목표는 기본적으로 반응형웹을 기본바탕으로 삼아 해상도에 따라 컨텐츠들이 중앙에 배치되도록 만들었고

css파일의 경우 효율적으로 만들기위해 재사용율을 높여 분리하여 다시 쓸 수 있는 부분과 그렇지 않은 부분을 나누려고 노력했습니다.


## 화면구성

화면 구성의 경우 상단의 헤더, 메인, 하단의 푸터로 나눌 수 있고, 푸터의 경우 메인의 높이변동에도 변함없이 하단에 머물 수 있게 배치하였습니다.


## HTML 작성 시 신경 쓴 점

기본골자로 시맨틱 태그를 사용해 구분하기 쉽도록 코드를 작성하려고 노력했으며, 콘텐츠에 맞게 이름의 설정과 재사용 css를 위한 이름의 통일감을 주었습니다.


##  CSS 작성 시 신경 쓴 점

css 작성시 신경 쓴 것은 앞서말한 재사용율을 높이는 것과 반응형 웹을 만들기위한 정렬,

내용에 따라서는 미디어 쿼리를 사용하여 메인컨텐츠의 불필요한 공간이 남지않도록 작성하는 것에 중점을 두었습니다.


## 개선할 점

처음에 html 코드를 작성 할 때 제대로 된 계획을 세우는 것이 아닌, 틀만 잡고 곁가지에 살을 붙여나가는 식으로 작성하였는데 지금와서 생각해보니 굉장히 무모한 계획이라고 생각되었습니다.

그렇다보니 처음 어렴풋이 생각한 것과 다르게 나온 것이 많아 코드를 수정하고, 결과를 다시 확인하는 과정이 많았습니다. 이를 통해 나의 기초가 생각보다 빈약하고 보충이 필요하다는 것을 느꼈습니다.

또한 css작성과 html의 작성에도 너무 특정 속성을 남발하여 불필요한 소스를 생성한 것은 아닌가 라는 생각도 들고 있습니다.

불필요한 소스가 생성되면 코드를 알아보는 것에 어려움이 생기고 추후 유지 보수에 어려움이 생길 것이기 때문에 이를 위해서라도 기초를 보강해 효율성을 높히고 싶습니다.

Readme를 작성하는데 용어에 대한 혼동이 있어 코딩의 기초뿐만 아니라 개념에 대한 기초 또한 바로 잡을 필요성을 깨달았습니다
